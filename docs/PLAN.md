## 1. Scope decisions (eigenaar)

- **Wie:** de eenmanszaak levert een SaaS. Klanten betalen via **Mollie**; dat is de enige omzetstroom.
- **Doel:** btw-aangifte (omzetbelasting) volledig voorbereiden; indienen blijft handmatig via Mijn Belastingdienst Zakelijk â€” er is geen publieke indien-API (Digipoort = PKIoverheid-certificaat voor softwareleveranciers, out of scope).
- **Btw-plichtig, gĂ©Ă©n KOR; aangifte per kwartaal.**
- **Dubbel boekhouden (double-entry):** elke transactie is een Journaalpost met debet-/creditregels op een rekeningschema (chart of accounts); som(debet) == som(credit) is een harde invariant. De UI houdt het simpel met sjabloonflows die gebalanceerde posten genereren; balans en winst & verlies komen gratis mee. Zie DATA-MODEL.md Â§1.
- **Inkomsten = automatische Mollie-import.** Boekhond haalt betaalde Mollie-payments (en refunds) op via de Mollie API (`howl.request`, API-key uit `.env`) en boekt ze als omzet (21% â†’ rubriek 1a) tegen debiteuren/bank. Idempotent per `mollie_payment_id`. **Boekhond maakt zelf geen verkoopfacturen:** de SaaS-app genereert en bewaart de genummerde btw-verkoopfacturen (PDF) al; de fiscale bewaarplicht voor uitgaande facturen ligt daar. Boekhond boekt alleen de omzet.
- **Uitgaven = terugkerend + handmatig.** Vaste maandabonnementen (Resend, Claude/Anthropic, OpenAI â€” digitale diensten van buiten de EU, btw verlegd naar mij â†’ rubriek 4a + 5b) worden per sjabloon maandelijks automatisch geboekt. Daarnaast een **handmatige knop** om een losse inkoopfactuur toe te voegen. Inkoopbewijs (PDF's van die diensten) komt binnen via een open uploadveld of de `data/import/`-map â†’ import-inbox.
- **Hosting: Docker-container in een VM, alleen bereikbaar via LAN/VPN.**
- **Geen app-login/wachtwoord.** Single-user; de enige toegangsgrens is het netwerk (LAN/VPN-only, achter een reverse proxy). Bewuste keuze â€” een in-app wachtwoord zonder TLS/KDF gaf geen echte bescherming; niet opnieuw toevoegen (ARCHITECTURE Â§5.1).
- **Opslag: DSON-bestanden** (`dson` stdlib) â€” eigenaarskeuze, on-theme; geen database (Doge heeft geen driver; sqlite-via-subprocess afgewezen). Store-laag blijft een seam.
- **Frontend: server-rendered HTML + Dogescript** (https://github.com/dogescript/dogescript, compileert naar JS) voor client-side gedrag. **Geen issues aanmaken op dogescript** â€” "we have what we get"; gaten vullen met gewone JS.
- **IB-aangifte (inkomstenbelasting) is out of scope** â€” het jaaroverzicht (omzet/kosten per categorie) is het enige gebaar die kant op.

## 3. Language gaps

Open taalgaten:

- **doge#70 â€” geen sequence-repeat `*`** (`Str * Int`, `List * Int`): `"a" * 5` geeft een `TypeError` i.p.v. `"aaaaa"` (Python-stijl). Louter ergonomie, laag prioriteit; alleen geraakt in een test (een grote `Bytes` bouwen). Geen productie-stopgap nodig â€” de test verdubbelt een `Bytes` (`b = b + b`). Ticket op DogeLanguage/doge.
- **doge#71 â€” `howl.accept` geeft geen peer-adres:** een loopback-only endpoint kan daardoor niet op de remote-IP filteren, alleen op een secret. `/internal/run-recurring` (Phase 5c) guard't daarom **token-only** via `crypto.same` op `INTERN_TOKEN` â€” `# stopgap for doge#71` in `app/handlers/intern_h.doge`. **Wordt gefixt in v0.3.4** (next release): bij de upgrade naar Doge v0.3.4 de stopgap vervangen door een loopback-peer-check (naast de token) en de `# stopgap`-comment verwijderen.

Gesloten in **v0.3.3** (waren open in 0.3.2):

- ~~**doge#67 â€” string-escapes kennen geen `\r`**~~ **GESLOTEN (0.3.3):** `"\r"` werkt nu. De CRLF-stopgap in `web/http.doge`/`tests/test_http.doge` is vervangen door `so CRLF = "\r\n"`.
- ~~**doge#68 â€” `Bytes` heeft geen substring-search**~~ **GESLOTEN (0.3.3):** `bytes.find(sub)` â†’ Int-offset of `-1`, `bytes.split(sep)` â†’ `List<Bytes>`, `bytes.contains(sub)` â†’ Bool. `web/multipart.doge` gebruikt deze native; er is nooit een byte-scan-stopgap geschreven. (`bytes.index` bestaat niet.)

De stdlib dekt verder alles wat fase 0â€“1 nodig had: binaire sockets (`howl.recv_bytes`/`send_bytes`), `crypto` (sha256/hmac/token/same), de volledige HTTP-client (`howl.request` met headers/methods), en base64/hex (`bytes.b64()`, `str.from_b64()`/`from_hex()`).

Loop je tegen een nieuw gat aan, dan geldt Hard Rule 10: ticket op DogeLanguage/doge + de kleinste `# stopgap for doge#NN`-workaround, en hier Ă©Ă©n regel bij. **Bewust *niet* gevraagd** aan de taal: SQLite/DB-driver (DSON-bestanden zijn de juiste maat), date-arithmetic (`lib/datum.doge` hoort in userland), TLS-server (reverse proxy is het standaardantwoord), HTML-templating (userland).
